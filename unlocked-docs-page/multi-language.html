Multi-language system

One of the biggest selling points of Arix Theme is the multi-language system of Arix Theme. The system makes it possible that people can pick their own language in the account settings.

How to add the language pack?

The first and most simple way of adding translations to your panel is by using our translation pack. After download the translation pack, drag and drop the resources folder into your Pterodactyl folder. Then run:

cd /var/www/pterodactyl
export NODE_OPTIONS=--openssl-legacy-provider
yarn build:production

How to add my own translation?

Adding your own translation is straightforward.

Navigate to /resources/lang/. Copy the folder en and rename it to the language code. Example: rename it to nl for Dutch.

Translate all files inside arix. Example: /resources/lang/nl/arix/

Then edit the file /resources/scripts/components/dashboard/forms/AppearanceWrapper.tsx

const languages = [
    { key: 'en', value: 'English' },
];

Change to:

const languages = [
    { key: 'en', value: 'English' },
    { key: 'nl', value: 'Dutch' },
];

Then run:

cd /var/www/pterodactyl
export NODE_OPTIONS=--openssl-legacy-provider
yarn build:production

How to add translations to addons?

Much of Pterodactyl is now available in multiple languages because of Arix Theme. Addons still require translation manually.

Adding translations to addons requires programming experience.

Navigate to /resources/lang/en/arix/server and create a file like plugins.php.

<?php
return [
    'example' => 'This is an example value',
    'example-group' => [
        'group-child' => 'This is an example group.',
    ],
];

Use translations inside React files:

import React from 'react'
import { useTranslation } from 'react-i18next'

const ExampleContainer = () => {
    const { t } = useTranslation('arix/server/plugin')

    return (
        <div>
            {t('example')}
            {t('example-group.group-child')}
        </div>
    )
}

export default ExampleContainer

Finally run:

cd /var/www/pterodactyl
export NODE_OPTIONS=--openssl-legacy-provider
yarn build:production

We do not provide support for addon translation.
